# Quiver Improvements

This registry records reusable improvements discovered while developing or
running Quiver. Edit it through `./qi` so `INDEX.json` stays derived from item
files.

Lifecycle:

- `proposed`: observed but not accepted
- `accepted`: worth implementing
- `implemented`: code or process changed
- `verified`: confirmed by a real Quiver run
- `rejected`: not worth pursuing

`implemented` is not terminal. Move an item to `verified` only after the change
has survived a concrete run or regression test.
